## 1. AdSense 탈락 대비 - 콘텐츠 강화 계획

### 문제점
Google AdSense는 "도구만 있는 사이트"를 "가치가 낮은 콘텐츠"로 판단할 수 있음.

### 해결책: 각 도구 페이지에 설명 콘텐츠 추가

#### 추가할 섹션 구조
```html
<!-- 도구 UI 아래에 추가 -->
<section class="mt-12 prose dark:prose-invert max-w-none">
  <h2>Base64란?</h2>
  <p>Base64는 바이너리 데이터를 텍스트로 변환하는 인코딩 방식입니다...</p>

  <h2>사용 방법</h2>
  <ol>
    <li>변환할 텍스트를 입력합니다</li>
    <li>인코딩 또는 디코딩 버튼을 클릭합니다</li>
    <li>결과를 복사합니다</li>
  </ol>

  <h2>자주 묻는 질문</h2>
  <h3>Base64는 암호화인가요?</h3>
  <p>아니요, Base64는 인코딩이지 암호화가 아닙니다...</p>
</section>
```

#### 각 도구별 추가할 콘텐츠

| 도구 | 추가 콘텐츠 |
|------|------------|
| Base64 | Base64란?, 사용 사례, URL-safe Base64 차이 |
| URL 인코딩 | URL 인코딩이 필요한 이유, 특수문자 처리 |
| JWT | JWT 구조 설명, Header/Payload/Signature, 보안 주의사항 |
| UUID | UUID 버전별 차이 (v1, v4, v5), 사용 사례 |
| 해시 | 해시 함수란?, MD5 vs SHA 비교, 보안 용도 |
| JSON | JSON 문법, 흔한 오류, JSON vs XML |
| 색상 변환 | HEX/RGB/HSL 설명, 색상 이론 기초 |
| Timestamp | Unix 시간이란?, Y2K38 문제, 타임존 |
| Regex | 정규표현식 기초, 자주 쓰는 패턴, 치트시트 |
| Lorem Ipsum | Lorem Ipsum 역사, 왜 사용하는지 |
| 연봉 계산 | 4대보험 설명, 세금 계산 방법, 연봉 협상 팁 |
| 대출 계산 | 원리금균등 vs 원금균등, 금리 종류 |
| BMI | BMI 계산 공식, 한계점, 건강 기준 |
| D-day | 날짜 계산 방법, 윤년 처리 |
| 글자수 | UTF-8 바이트 계산, SNS 글자수 제한 |
| 단위 변환 | 평 ↔ ㎡ 역사, SI 단위 |
| QR 코드 | QR 코드 구조, 오류 정정 레벨 |
| 모니터 테스트 | 데드픽셀 vs 스턱픽셀, 보증 정책 |

## 2. 광고 배치 계획 (4개 광고)

### 현재 구조
```
┌─────────────────────────────┐
│          HEADER             │
├─────────────────────────────┤
│      [광고 - 상단]          │
├─────────────────────────────┤
│                             │
│         도구 영역           │
│                             │
├─────────────────────────────┤
│      [광고 - 하단]          │
├─────────────────────────────┤
│          FOOTER             │
└─────────────────────────────┘
```

### 제안 구조 (4개 광고)
```
┌─────────────────────────────────────────┐
│               HEADER                    │
├─────────────────────────────────────────┤
│            [광고 - 상단]                │
├───────┬─────────────────────────┬───────┤
│       │                         │       │
│ [좌측]│       도구 영역         │[우측] │
│ 광고  │                         │ 광고  │
│       │                         │       │
├───────┴─────────────────────────┴───────┤
│            [광고 - 하단]                │
├─────────────────────────────────────────┤
│               FOOTER                    │
└─────────────────────────────────────────┘
```

### 장단점 분석

#### 장점 ✅
| 항목 | 설명 |
|------|------|
| 수익 증가 | 광고 4개 = 수익 2~3배 가능 |
| 사이드바 CTR | 사이드바 광고는 스크롤해도 보임 (sticky) |
| 다양한 광고 유형 | 배너, 사각형, 세로형 등 다양하게 배치 가능 |

#### 단점 ❌
| 항목 | 설명 |
|------|------|
| UX 저하 | 도구 영역이 좁아짐 |
| 모바일 대응 | 모바일에서 좌우 광고 처리 필요 |
| 로딩 속도 | 광고 많으면 페이지 느려짐 |
| AdSense 정책 | 광고가 콘텐츠보다 많으면 정책 위반 가능 |

### 권장 전략

#### Phase 1: 승인 후
- 상단 + 하단 (2개)만 유지
- 안정적으로 수익 확인

#### Phase 2: 트래픽 증가 후
- 좌측 사이드바 추가 (sticky)
- 총 3개

#### Phase 3: 최적화
- 우측 사이드바 테스트
- A/B 테스트로 수익 비교
- 너무 많으면 다시 줄이기

### 사이드바 광고 CSS
```css
/* Sticky 사이드바 광고 */
.sidebar-ad {
  position: sticky;
  top: 80px; /* 헤더 높이 */
  width: 160px; /* 또는 300px */
}

/* 모바일에서는 숨김 */
@media (max-width: 1200px) {
  .sidebar-ad {
    display: none;
  }
}
```

### 레이아웃 변경 필요사항
```html
<!-- 현재 -->
<main class="container-main">
  <div class="ad-container">상단 광고</div>
  <div class="tool-content">도구</div>
  <div class="ad-container">하단 광고</div>
</main>

<!-- 변경 후 -->
<main class="container-main flex">
  <aside class="sidebar-ad hidden xl:block">좌측 광고</aside>
  <div class="flex-1">
    <div class="ad-container">상단 광고</div>
    <div class="tool-content">도구</div>
    <div class="ad-container">하단 광고</div>
  </div>
  <aside class="sidebar-ad hidden xl:block">우측 광고</aside>
</main>
```
