What are Line Endings?

Line endings (also known as line breaks or newline characters) are special characters that mark the end of a line in text files. Different operating systems have historically used different line ending conventions, which can cause issues when opening files created on one OS in another. This tool helps you convert between different line ending formats easily.

Line Ending Types

  • LF (Line Feed, \n): Standard line ending used by Unix, Linux, and modern macOS.
  • CRLF (Carriage Return + Line Feed, \r\n): Line ending used by Windows.
  • CR (Carriage Return, \r): Line ending used by classic Mac OS (version 9 and earlier).

How to Use

  1. Paste or type the text you want to convert in the input area.
  2. Click the "Detect Format" button to identify the current line ending format.
  3. Click the desired conversion button (LF, CRLF, or CR).
  4. Copy the converted result using the "Copy" button.

Frequently Asked Questions

Q: Why do I need to convert line endings?

Line ending mismatches can cause issues when uploading files from Windows to Linux servers, collaborating on Git repositories, or running shell scripts. For example, shell scripts with Windows line endings may fail to execute, or Git diffs may show unnecessary changes.

Q: What is "Mixed" format?

Mixed format indicates that a file contains multiple types of line endings. This often happens when a file has been edited on different operating systems. It's recommended to normalize to a single format for consistency.

Q: Which format should I use?

For modern projects, LF is generally recommended. Git defaults to LF, and most code editors and IDEs handle LF well. Use CRLF only if you're working on a Windows-only project or with legacy Windows applications that require it.
