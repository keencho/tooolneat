What is YAML ↔ JSON Converter?

YAML and JSON are widely used data serialization formats. YAML is human-readable and commonly used for configuration files, while JSON is the standard for API communication and data exchange. This tool allows you to easily convert between these two formats.

How to Use

  1. Enter data in either the YAML area (left) or JSON area (right).
  2. Click 'YAML → JSON' or 'JSON → YAML' button to convert.
  3. When auto-detect is enabled, input format is recognized automatically.
  4. Choose indentation size (2 or 4 spaces) for output formatting.
  5. Use the Copy button to copy results to clipboard.

Frequently Asked Questions

Q: What's the difference between YAML and JSON?

YAML uses indentation to represent structure and supports comments. JSON uses braces and brackets for structure and doesn't support comments. YAML is more concise and readable, while JSON is stricter and faster to parse.

Q: How are multiline strings converted?

YAML multiline strings (| or >) are converted to escaped strings with \n in JSON. Conversely, escaped newlines in JSON are properly handled when converting to YAML.

Q: What happens to comments during conversion?

YAML comments (#) are removed when converting to JSON because JSON doesn't support comments.
