What are HTML Entities?

HTML entities are special codes used to represent characters that have special meaning in HTML or characters that cannot be easily typed on a keyboard. They start with an ampersand (&) and end with a semicolon (;). For example, &lt; represents the less-than sign (<), and &amp; represents the ampersand (&) itself. These entities ensure that special characters are displayed correctly in web browsers without being interpreted as HTML code.

How to Use

  1. Select "Encode" or "Decode" mode depending on your needs.
  2. Choose encoding options: encode only special characters or all characters.
  3. Select the entity format: Named (&amp;), Numeric (&#38;), or Hex (&#x26;).
  4. Enter your text in the input field on the left.
  5. With "Live" mode enabled, the result updates automatically. Otherwise, click the Convert button.
  6. Use the "Copy" button to copy the result to your clipboard.

Frequently Asked Questions

Q: Why do I need HTML entities?

In HTML, characters like <, >, and & have special meanings - they define tags and entities. To display these characters as text instead of being interpreted as code, you must convert them to entities. Without this conversion, your content might break the page layout or cause security vulnerabilities.

Q: What is the difference between Named and Numeric entities?

Named entities like &amp; use human-readable names, making them easier to understand in source code. Numeric entities like &#38; use the character's Unicode code point. While named entities are more readable, not all characters have named versions. Numeric entities can represent any Unicode character, providing universal coverage.

Q: How do HTML entities help prevent XSS attacks?

Cross-Site Scripting (XSS) attacks occur when malicious scripts are injected into web pages. By encoding user input as HTML entities before displaying it, you prevent scripts from executing. For example, <script> becomes &lt;script&gt;, which is displayed as text rather than executed as code.
