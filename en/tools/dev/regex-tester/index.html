Regex Cheat Sheet

Character Classes

. - Any character (except newline)
\d - Digit [0-9]
\D - Non-digit
\w - Word character [a-zA-Z0-9_]
\W - Non-word character
\s - Whitespace
\S - Non-whitespace

Anchors

^ - Start of string/line
$ - End of string/line
\b - Word boundary
\B - Non-word boundary

Quantifiers

* - 0 or more
+ - 1 or more
? - 0 or 1
{n} - Exactly n
{n,} - n or more
{n,m} - Between n and m

Groups

(abc) - Capture group
(?:abc) - Non-capture group
(?<name>abc) - Named group
\1 - Backreference

Lookaround

(?=abc) - Positive lookahead
(?!abc) - Negative lookahead
(?<=abc) - Positive lookbehind
(?<!abc) - Negative lookbehind

Special Characters

[abc] - Character set
[^abc] - Negated set
[a-z] - Character range
a|b - Alternation (OR)
\ - Escape character

What is Regex Tester?

A Regex Tester (Regular Expression Tester) is a tool that helps developers create, test, and debug regular expressions in real-time. Regular expressions are powerful patterns used for searching, matching, and manipulating text. They are essential for tasks like form validation, data extraction, search and replace operations, and parsing log files. This tool provides instant visual feedback, capture group highlighting, and a built-in cheat sheet.

How to Use

  1. Enter your regular expression pattern in the Pattern field (without the surrounding slashes).
  2. Select the appropriate flags: g (global), i (case-insensitive), m (multiline), s (dotall), u (unicode).
  3. Type or paste your test string in the Test String area to see matches highlighted instantly.
  4. Review the Match Results section to see all matches, their positions, and captured groups.
  5. Enable "Replace Mode" to test string replacement using capture group references ($1, $2, etc.).

Frequently Asked Questions

Q: What do the regex flags (g, i, m, s, u) mean?

g (global) finds all matches instead of stopping at the first one. i (case-insensitive) ignores uppercase/lowercase differences. m (multiline) makes ^ and $ match the start and end of each line. s (dotall) makes the dot (.) match newline characters. u (unicode) enables full Unicode support for patterns and character classes.

Q: What are capture groups and how do I use them?

Capture groups are created using parentheses () in your pattern. They "capture" the matched text so you can reference it later. In replacements, use $1 for the first group, $2 for the second, and so on. Named groups (?<name>pattern) can be referenced by name. Non-capturing groups (?:pattern) group without capturing.

Q: Why isn't my regex matching as expected?

Common issues include: forgetting to escape special characters (like . * + ? [ ] { } etc.), not enabling the global flag for multiple matches, case sensitivity issues (use the i flag), or multiline text issues (use the m flag). Also check for invisible characters like different types of whitespace or line endings.
